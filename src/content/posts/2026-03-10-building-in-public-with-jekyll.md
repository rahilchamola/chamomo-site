I could have used a template. Literally hundreds of polished Jekyll themes exist, and a few of them are genuinely beautiful. But I didn't want beautiful — I wanted *mine*.

This site is built from scratch. Every layout, every Sass variable, every Liquid template. Not because I'm a frontend developer (I'm not), but because I believe you learn systems by building them, not by consuming them.

## Why Jekyll

Jekyll hits a sweet spot that's hard to find elsewhere. It's simple enough that you can understand the entire build pipeline in an afternoon, but flexible enough that you never hit a ceiling. No database. No JavaScript framework. Just Markdown files that become HTML.

For someone with ADHD, this matters more than you'd think. The friction of writing should be zero. Open a file, write in Markdown, save. Done. No CMS login, no rich text editor fighting you, no deploy pipeline to babysit.

## The Learning Model

I'm treating this site as a learning vehicle. Every time I need something new — a tag cloud, a related posts section, a new collection type — I build it. The constraint of doing it myself means I actually understand what's happening.

This is the same approach I take with AI tools. Don't just use them — understand the system well enough to remix it. The remix is where the interesting stuff happens.

## What's Next

I'll be documenting the build process here as I go. CSS architecture decisions, content organization, deployment setup. If you're thinking about building your own site, maybe my mistakes will save you some time.

The source is the site. The site is the source. Let's see where it goes.
